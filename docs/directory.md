# ディレクトリ構成案

## ルート直下
- `docs/` : 要件定義や仕様書などのドキュメントを配置
- `src/` : 実装コードを格納するトップディレクトリ
- `tests/` : 自動テスト関連のコードとデータ
- `tools/` : 開発補助スクリプトやユーティリティ
- `package.json` / `tsconfig.json` など : 使用するフレームワークやビルドツールの設定ファイル

## docs/
- `requirements.md` : シナリオ要件（既存ファイルを移動予定）
- `directory.md` : 本ドキュメント
- `design/` : 画面遷移図、UI モックなどの設計資料

## src/
- `app/` : アプリケーション本体
  - `components/` : UI コンポーネント（React コンポーネント等）
  - `scenes/` : フェーズ単位の画面／ロジック
  - `state/` : 状態管理（Redux、Zustand などを想定）
  - `services/` : API やログ取得の抽象化
  - `assets/` : 画像・音声・翻訳ファイル
- `index.html` / `main.tsx` : エントリーポイント

## tests/
- `unit/` : 単体テスト
- `integration/` : シナリオ全体の結合テスト
- `e2e/` : ユーザー操作を想定した E2E テスト

## tools/
- `scripts/` : データ生成やビルド補助スクリプト
- `ci/` : CI 用の設定やワークフローファイル

## 補足
- 実際の実装に合わせてディレクトリを追加・再編することを想定
- Web ベースのインタラクティブ教材を前提とした構成
- 今後 `docs/requirements.md` を更新・拡充しながら、`src/` 以下の詳細構成を詰めていく
