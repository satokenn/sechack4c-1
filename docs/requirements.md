# 概要

## プロジェクト名
- セキュリティ・インシデント対応シミュレーター（仮称）
- 通称：セキュリティ版スクラッチ

## プロジェクトの目的
セキュリティに興味を持ち始めた初学者を対象とし，複雑な技術詳細ではなく「インシデント発生時に，なぜ，何を，どのような順序で対応すべきか」という基本概念とプロセスをゲーム形式で学習できるようにする．

## 本書の位置づけ
プロジェクト内で開発する複数シナリオのうち，コアとなる「シナリオ2：謎の通信を遮断せよ」の要件を定義する．

# ターゲットユーザー
- セキュリティに興味を持ち始めた学生または社会人
- 専門知識は深くないが概念を学びたい人
- スクラッチ等のビジュアル教材に親和性がある人

# シナリオ概要：「謎の通信を遮断せよ」

## あらすじ
標的型攻撃メールにより鈴木さんがマルウェア感染。プレイヤーはセキュリティ担当者としてアラートから感染を検知し，隔離→分析→横展開調査を進めて終息させる。

## 学習目標
- **検知**：アラートがインシデント対応の起点になることを理解する。
- **隔離**：最優先行動が被害拡大防止であると体験する。
- **調査**：原因特定と影響範囲の把握が不可欠だと学ぶ。

## 主要登場人物
- **鈴木さん（NPC）**：経理部所属。標的型メールを開封してしまう。
- **プレイヤー（主人公）**：セキュリティ担当者。アラート対応を担う。
- **メンター（NPC）**：上司または先輩。状況を解説し選択を促す。

# 機能要件（ゲームフロー）
シナリオは 6 フェーズで構成される。

## フェーズ1：やらかし（導入）
- **形式**：紙芝居形式の自動再生ムービー。
- **流れ**：鈴木さんが「請求書.zip」を実行し，見た目に変化なく感染が進行。マルウェアが C&C 通信を開始。
- **UI**：再生／スキップボタン，吹き出し等の台詞表示。

## フェーズ2：検知（アラート）
- **形式**：初回インタラクション。
- **UI**：監視ダッシュボードが表示され，高優先度アラートが点滅。
- **操作**：アラートをクリックして詳細表示。
- **結果**：詳細ウィンドウに発生元 PC と通信先 IP を表示し，隔離フェーズへ誘導。

## フェーズ3：初動対応（隔離）
- **形式**：単一選択。
- **選択肢**：
	1. 物理的隔離（正解）
	2. リモート隔離（失敗）
	3. 電話連絡（失敗）
	4. 監視継続（失敗）
- **結果**：正解はフェーズ4へ。失敗時はカットシーン後リトライ。

## フェーズ4：調査・分析
- **形式**：ドラッグ＆ドロップ。
- **操作**：
	- 「不審なファイルのスキャン」でマルウェア本体を特定。
	- 「メールログの解析」で侵入経路を特定。
- **UI**：調査対象アイコンとツールボックスを表示。

## フェーズ5：横展開調査と駆除
- **形式**：検索 UI とボタン操作。
- **操作**：
	- 全社通信ログ検索で横展開状況を確認。
	- 全社メールログ検索で同じ攻撃メールの配信先を確認。
	- メール一斉削除とマルウェア駆除ボタンを実行。
- **結果**：調査結果を表示し，対処アニメーションを再生。

## フェーズ6：まとめ
- **形式**：結果表示カットシーン。
- **UI**：「STAGE CLEAR」と学びの箇条書きを表示。
- **結果**：メンターが教訓を総括し，シナリオ終了。

# 非機能要件（抜粋）
- **プラットフォーム**：Web ブラウザ上で動作すること。
- **UI/UX**：専門用語はメンターが補足し，操作はクリックとドラッグ＆ドロップのみ。
- **リプレイ性**：失敗時はフェーズ3から即座にリトライ可能。
